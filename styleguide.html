Color

Hover/active derivations are produced inline with color-mix(in srgb, var(--accent) N%, transparent) rather than precomputed tokens — every hover state computes from the base accent at 6–60% alpha.

Duration ramp

Same easing, three durations

The same power3.out at three durations. Lower-bound 0.4s for UI feedback. Upper bound around 1.4s for hero reveals — past that and the user is waiting on the page.

0.4s — short
0.9s — medium
1.4s — long
0.4s — UI feedback, mini lift on hover·0.6–0.9s — entrance defaults·1.0–1.4s — hero word-reveals, ornament stamps·1.4s+ — programmatic scrolls only, never tweens (Lenis is currently disabled site-wide; native smooth-scroll falls through)

Scrub vs toggle

Scroll past — see the difference

Scrub binds a tween's progress to scroll position 1:1. scrub: 0.6 adds a 0.6s smoothing lag — the eased flavor used on parallax moments like the chapter-stage scroll-out, TBR drift, and portrait drift. The toggle version plays once at the entry threshold and ignores the rest. Book-page sections do not scrub — they animate on enter/exit only, so the user's scroll feels free.

scrub: 0.6 — eased
Q
toggleActions: play none none reverse
Q
scrollTrigger: { trigger, start: 'top 80%', end: 'bottom 30%', scrub: 0.6 }

Reduced motion

prefers-reduced-motion: reduce

When the OS-level reduce-motion flag is set, the system disables: page curtain, ink cursor, magnetic affordance, scroll-driven scrubs, word-mask reveals, image fade-in, reading-progress bar, and headline exhale. CSS hovers and theme transitions remain. The toggle below simulates the state on this page only — site-wide overrides require the OS-level setting.

§ VI Public API window.HL — what's exposed, what it does.

HL.motion

From js/motion.js

Initialised on every page that loads js/motion.js. Components can read prefersReducedMotion for guards or call scrollTo when they want a programmatic jump that rides Lenis easing.

Member Kind Description
HL.motion.prefersReducedMotion boolean (read-only) True when the OS-level prefers-reduced-motion: reduce media query matches. Components should branch on this before tweening.
HL.motion.isCoarsePointer boolean (read-only) True on touch devices and other coarse-pointer inputs. Used to suppress Lenis, the ink cursor, and magnetic affordance.
HL.motion.lenis Lenis instance | null Currently disabled — initLenis() returns null early because the 1.15s duration / 0.085 lerp interpolation produced perceptible input-lag on long pages. The hook is preserved so re-enabling is one-line, and any code that already guards on lenis && … continues to work. When live, this is the active Lenis instance (or null on coarse pointer / reduced motion / CDN failure).
HL.motion.scrollTo(target, options?) function Scroll to an element, selector, or absolute Y. With Lenis disabled, calls native scrollIntoView({behavior:'smooth'}). If Lenis is re-enabled, uses lenis.scrollTo with default duration 1.4 + cubic-out easing; options forward to Lenis ({ duration, offset, easing }).
HL.splitWords(el) function DIY word-splitter. Wraps each word in <span class="word"><span class="word-inner">…</span></span>. Preserves nested elements (<em>, accent spans). Returns the array of inner spans — tween those, not the outers.
HL.makeDrawable(el) function Adds .draw-line to an element so it starts at scaleX(0) with left transform-origin. Pair with gsap.to(el, { scaleX: 1 }) for the in-draw effect. Used on dashes and dividers throughout.
